## 🎯 Objetivo de la Práctica
Implementar un flujo de trabajo utilizando Git y GitHub Desktop para la gestión de versiones de un documento. Se aplica el uso de ramificaciones (branching), confirmaciones (commits) y fusiones (merging) para garantizar la trazabilidad del proyecto.

## 🛠️ Herramientas Utilizadas
* **Git:** Sistema de control de versiones distribuido.
* **GitHub Desktop:** Interfaz gráfica para la gestión del repositorio.
* **Microsoft Word:** Editor para la creación del contenido técnico.

## 🌿 Estructura de Ramas
Para este proyecto se utilizó una estrategia de ramificación basada en características (feature branching):

1.  **main:** Rama principal que contiene la versión final y estable del documento.
2.  **desarrollo:** Rama para el cuerpo principal y contenido técnico.
3.  **conclusion:** Rama para el cierre del trabajo y lecciones aprendidas.

## 🚀 Proceso de Trabajo
1.  **Inicialización:** Creación del repositorio y primer commit con la estructura básica.
2.  **Desarrollo Modular:** Creación de ramas independientes para cada sección del informe.
3.  **Versionado:** Registro de cambios mediante commits con mensajes descriptivos siguiendo el taller sugerido.
4.  **Integración:** Fusión de las ramas (Merge) hacia la rama `main` y resolución de conflictos de versión.
5.  **Sincronización:** Publicación de los cambios en el repositorio remoto (Push origin).
